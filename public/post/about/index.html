The Home Depot - Open Source Software Manifesto

We are developers who want to support and sustain open source software.

We value:

  • Respect
  • Collaboration
  • Healthy debates

We believe that all projects should:

  • Have a readme file
  • Have a contributing file
  • List all core team members in the readme file
  • Have a license file
  • Have a changelog
  • Follow semantic versioning
  • Tag all major releases
  • Provide documentation

How to get started?

Ask yourself the following questions

  1. Is what I’m working on something that would benefit the wider development community?
  2. Do I feel comfortable that work I’m doing does not include trade secrets?
  3. Have you written the code as a pair, or have a code reviewer?
  4. Why are you still asking questions, push some code!

Tactical Process

  1. Ensure your manager/team lead approves the effort
  2. Request open source submission approval from Chris Black
  3. Push the code
